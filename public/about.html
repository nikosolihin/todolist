Homework 8

  • What server framework are you using?

    For the server I am using express, a web application framework for node.js.

  • Is it programmatic or template-based?

    Express is more programmatic out of the box, but there are a few template engines that can be paired with it. Furthermore, express makes no assumption in terms of your application structure.

  • On a scale of 1-5 (1=easy), rate the learning curve of your chosen framework. Justify your answer.

    I would say that the learning curve of express is 2. It is a very lean framework that provides basic RESTful routing interface out of the box.

  • On a scale of 1-5 (1=easy), rate the difficulty of converting your app to the framework. Justify your answer.

    Converting my app to the backend framework was not too difficult, 2 out of 5. However, since express has no notion of a database, I went through some trial and errors setting up a third-party node module to interface with my MongoDB database at nodejitsu. Additionally, since express has no notion of application design pattern, most of my time was spent doing the frontend logic using backbone.

  • Would you consider using the framework again?

    Yes, I would. I like the simplistic nature of express and the fact that it is a RESTful framework.


Homework 9

  • How is your server implemented? RESTfully or Web Services?

    RESTfully using express as described above.

  • If it’s RESTful, provide the following information:

    • Resource identification - what are the resource abstractions?

      The resource is only the tasks in the To-Do List.

    • URI design - what do your URIs look like?

      Retreive all tasks:
GET http://cis658-todolist.nodejitsu.com/tasks

Retreive task with ID#:
GET http://cis658-todolist.nodejitsu.com/tasks/:id

Creating a new tasks:
POST http://cis658-todolist.nodejitsu.com/tasks

    With request body:    
    {
        "name": "Study for exam"
    }

Updating the status of task with ID#:
PUT http://cis658-todolist.nodejitsu.com/tasks/:id

    With request body:    
    {
        "name": "Study more",
        "status": "completed"
    }

Deleting a task with ID#:
DELETE http://cis658-todolist.nodejitsu.com/tasks/:id

    • Interaction semantics - do all four HTTP verbs apply to all resources?

      Yes. GET, POST, PUT, and DELETE apply.

    • Resource relationships - how are resources linked?

      The resources are not linked since this is a very simple application. However, if the application has the capability of handling different users, there will be "state transfer" links/relationships between tasks and the users resource abstraction.

    • Data representation - XML, JSON, etc.

      JSON.

  • Did you use a framework? If so, which one?

    Yes. Express as described in homework 8.


Homework 10

  • Which PaaS provider did you choose for your application (if it’s not obvious from the URL)?

    NodeJitsu

  • What was the most challenging aspect of the transition to a cloud-based platform?

    For me, it was getting node to communicate with Nodejitsu's MongoDB, but it was probably because I have not done much with MongoDB before.

  • Did your PaaS provide make writing the code any easier, for example by providing useful utility APIs?

    Although not necessarily utility API, I did find nodejitsu's CLI to be quite helpful for deploying and debugging my application.
